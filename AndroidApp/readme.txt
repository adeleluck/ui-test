Для того, чтобы настроить конфигурацию appium для тестирования конкретного приложения, необходимо:

1. Извлечь значения package и activity name, с помощью утилиты aapt, входящей в состав средств разработки android.

2. Для того, чтобы извлечь эту информацию, необходимо расположить файлы aapt и apk в одной папке, открыть эту папку в командной строке и дать команду aapt dump badging *имя нашего apk файла*.apk > dump.txt

Например: aapt dump badging myapp.apk > dump.txt

3. Появится файл dump.txt в этой папке, из которого необходимо извлечь информацию о пакете и main activity

В начале файла идёт информация о package, например в таком виде:

package: name='com.myapp.app'

Информация об appActivity находится в строке с примерно таким видом:

launchable-activity: 'com.myapp.app.MyActivity'

Значение забираем без кавычек.

Изменения вносят в файл BaseConfig.java, который является родительским классом для всех классов, выполняющих UI тесты.

Конфигурация из этого класса будет применяться перед запуском UI тестов, реализованных в дочерних классах.

4. Далее необходимо запустить эмулятор андроид (для visual studio идентификатор тот, который мы используем в программе, если эмулятор иной, то получаем его идентификатор и записываем его в программу)
